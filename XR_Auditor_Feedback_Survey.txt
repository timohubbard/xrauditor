[[Block:Phase 2: Safety & Procedural Setup]]

[[Question:TE,Essay]]
REQUIRED FOR ALL VR/XR STUDIES:
- Contraindication screening: Screen all participants for epilepsy, vestibular disorders, severe motion sickness, and pregnancy before participation.
- Standard battery: Administer a presence measure (IPQ, SUS Slater-Usoh-Steed, or SPES) plus a VR Familiarity 2-item scale in every study.

Please provide any feedback or comments on these requirements (Optional):

[[Question:TE,Essay]]
CONDITIONAL: IF the study uses artificial locomotion (any non-walking movement such as teleportation or continuous/redirected walking):
- Administer CSQ-VR (Cybersickness Questionnaire for VR) before and after the session, capturing Nausea, Disorientation, and Sopite subscales.
- Log the locomotion type used (teleportation / continuous / redirected walking).
- Researcher rates comfort level (Comfortable / Moderate / Intense) at 5-minute intervals.

Please provide any feedback or comments on these requirements (Optional):

[[Question:TE,Essay]]
CONDITIONAL: IF sessions are longer than 30 minutes:
- Schedule mandatory breaks every 20–30 minutes.
- Re-administer CSQ-VR after each break.
- Researcher monitors for signs of discomfort: pallor, sweating, postural instability.
- Document break timing and any adverse events in the session log.

Please provide any feedback or comments on these requirements (Optional):

[[Question:TE,Essay]]
CONDITIONAL: IF the study includes humanoid avatars or virtual agents:
- Measure the Humanness Index if photorealistic humanoids are used (uncanny valley assessment).
- Document avatar type: pre-scripted / researcher-controlled / autonomous.
- Add SPS (Social Presence Scale) to the measurement battery.

Please provide any feedback or comments on these requirements (Optional):

[[Question:TE,Essay]]
CONDITIONAL: IF the study involves high-risk populations or clinical samples:
- Use an enhanced contraindication checklist with researcher sign-off.
- Have a clinician on-call if recruiting mental health populations.
- Lower the session duration limit to 20 minutes.
- Pre-define stopping rules and a debrief protocol for adverse events.

Please provide any feedback or comments on these requirements (Optional):

[[Question:TE,Essay]]
CONDITIONAL: IF the study uses a multi-site or multi-lab design:
- Use identical HMD model across all sites where possible.
- If different HMDs must be used, document for each site: model, IPD range, tracking type, and refresh rate.
- Share a setup checklist and video walkthrough with each lab.
- Run a pilot and share pilot data across labs before full data collection begins.

Please provide any feedback or comments on these requirements (Optional):

[[Block:Phase 3: Ethics, Pre-Registration & Data Repositories]]

[[Question:TE,Essay]]
REQUIRED FOR ALL VR/XR STUDIES:
- Ethics approval: Obtain IRB/REC approval before data collection. Include a VR-specific risk section in the ethics application covering cybersickness, adverse events, and biometric data handling.
- Repository setup: Create an OSF project page and a GitHub repo before data collection and link both in the ethics application.

Please provide any feedback or comments on these requirements (Optional):

[[Question:TE,Essay]]
CONDITIONAL: IF the study is confirmatory / hypothesis-driven:
- Pre-register on OSF Registries or AsPredicted before data collection begins.
- The pre-registration must include: Hypotheses, Study design, Sample size justification (power analysis), Analysis plan, VR-specific exclusion criteria (HMD removal, boundary breach, sickness dropout, tracking loss)

Please provide any feedback or comments on these requirements (Optional):

[[Question:TE,Essay]]
CONDITIONAL: IF the study collects biometric or otherwise identifiable data:
- Ensure the consent form explicitly covers biometric data use.
- Anonymise or pseudonymise all data before archiving.
- Use a controlled-access repository: OSF controlled access or Zenodo restricted record.
- State Data Use Agreement (DUA) requirements clearly in the repository README.
- Ensure compliance with GDPR (EU) and/or HIPAA (US) as applicable.

Please provide any feedback or comments on these requirements (Optional):

[[Question:TE,Essay]]
CONDITIONAL: IF the VR environment code is shareable:
- Commit the environment code to GitHub before data collection.
- Tag the release version used in the study (e.g., v1.0.0-datacollection).
- Include in the repository: Build instructions, Full asset list, Engine version, SDK versions, Researcher setup notes.

Please provide any feedback or comments on these requirements (Optional):

[[Question:TE,Essay]]
CONDITIONAL: IF participant data can be shared openly:
- Upload the de-identified dataset with a codebook to OSF under CC BY 4.0 license.
- The codebook must define: Variable names and units, Sampling rates, Instrument subscales, VR-specific fields: HMD model, tracking type, locomotion method.
